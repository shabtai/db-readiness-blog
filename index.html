Database readiness · measurement

What a low-signal table costs an AI agent

Two independent properties of a table change an agent's cost, latency, and correctness — and each has a different fix.

A benchmark study from Natural‑Joints · we drove a Claude Code agent through a fixed question set and logged every query it issued.

When you point an AI agent at a database, it first has to figure out what your tables and columns mean, and which rules the data quietly follows. We varied two independent things about the schema and measured the effect on each question:

  1. Column names — do the identifiers carry meaning, or are they opaque (c0, MATNR, FIELD_017)?
  2. Column values — do the values follow a convention stated nowhere (mixed units, a join that double-counts, an amount in cents)?

The four metrics

AccuracyAnswer matches a known gold result. The only correctness metric; the other three measure effort.
Queries / qSQL statements the agent issues per question — its exploration effort. A legible schema needs fewer.
Time / qWall-clock seconds per question; the round-trips before it can answer.
Price / qModel API dollars per question — driven by tokens: the schema it reads + the queries it runs.

Aspect 1 — Column names (legibility)

How we strip the naming signal

We take a normal, valid database and degrade only the names — data left byte-for-byte identical — in two ways:

  • Obfuscated — every identifier becomes an opaque code (tables t_a1b2, columns c0, c1, c2…). The names carry no signal.
  • Low-signal — every identifier becomes a real but generic word (tables entity/record, columns id/value/status/amount). The names look meaningful but carry false signal.

Values like "Carniolan" are untouched in both — the agent must reverse-engineer each column by sampling rows. This isolates exactly what naming signal is worth.

Low-signal names are worse than full obfuscation. Generic-but-plausible names (entity, value, status) scored the same 80% accuracy as opaque codes but cost more — 14 vs 12 queries, 70s vs 57s, $0.168 vs $0.139 — because false signal lures the model into confident wrong assumptions, while opaque codes simply say "you have nothing, sample everything." The time gap is the sharpest signal: a low-signal schema took ~4.4× longer per question than the legible baseline (16s → 70s). A semantic layer over the same data restored accuracy and pulled time back to ~30s. Across models, the highlighted Opus-on-raw price ($0.269) is the costly alternative — the same 100% accuracy is reached for $0.050 by enriching and dropping to the cheapest model.

To reach 100% on the opaque schema, you can spend on the model or on the schema:

Frontier model (Opus) on raw opaque data: $0.269 / question.
Cheapest model (Haiku) + a brief: $0.050 / question — same accuracy.

≈ 5× lower price, identical correctness

A legible, documented schema lets a small model do what otherwise needed a frontier one — enrichment substitutes for model capability.

Brief vs. the full map

A semantic layer can be handed to the agent two ways, and they are not equivalent:

  • The full map (a single .md file) — the complete curated reference: every table's grain, keys, column meanings, and detected hazards in one document. Complete, but the fact that matters for a given question is buried among everything else.
  • The brief (question-scoped) — one fast model call reframes the question against the schema: it ranks the relevant tables, expands those, and attaches only the keys and hazards that bear on this answer, while still listing every table so nothing is silently excluded. It costs about one extra cent and a few seconds.

The data (Opus 4.8, on a value-reconciliation question — “total treatment spend, in dollars,” where one column is stored in cents and reconciles to a dollar column in another table):

Layer formAccuracyQueries/qPrice/q
Full map (.md dump)2/3 (67%)4.7$0.201
Brief (scoped)3/3 (100%)4.3$0.176

Here the brief is not just cheaper — it is more accurate. With the full map, Opus saw the cents convention and two distracting neighbours (a dose column, a separate rate table) and over-reasoned: Σ dose × cost ÷ 100 = $239.50 — wrong. The scoped brief surfaced only the cost↔price reconciliation, so it summed correctly: $54.50. Same model, same data — the only difference is which context the layer put in front of it.

That is the core lesson, and it cuts both ways: more context is not better context. The full map's value grows with how much you can't fit in a prompt; the brief's value grows with how easily the one decisive fact gets buried.

Aspect 2 — Column values (conventions)

Now the names are fine, but the values follow a rule that appears in no name — yet is recoverable from the data itself. We planted three and asked questions that hinge on each (wrong-answer counts over three model tiers — Haiku, Sonnet, Opus):

DeviationRecoverable from…Raw schema+ semantic layer
Mixed units (kg & g)a sibling unit column0/3 wrong0/3 wrong
Fan-out join (1:many)join cardinality1/3 wrong0/3 wrong
Amount stored in centscross-table reconciliation3/3 wrong0/3 wrong
Overall5/9 correct9/9 correct

The cents case is the sharp one: an integer column actually in cents, reconcilable against a dollar column in another table. Every model got it wrong 3/3 on the raw schema — summing it as dollars (a 100× error). None reconciled the two tables unprompted; the enricher had, so with the layer all three answered correctly.

Same accuracy — what it costs in time & budget

On these value questions the raw schema caps at 5/9 correct for every model — none reaches 100% unaided. Once you require correct answers you must enrich; the only remaining choice is which model. To reach 3/3 (100%):

Model + semantic layerAccuracyTime/qPrice/q
Haiku 4.5100%17s$0.037
Sonnet 4.6100%19s$0.088
Opus 4.8100%28s$0.176

Same correctness, very different price: the cheapest model reaches it for $0.037/q in 17s — about 5× cheaper and ~40% faster than the frontier model. Because correctness here requires the layer regardless of model, the gain is choosing the smallest model the layer can carry.

This is the difference between the two aspects: opaque names cost mostly budget (the agent grinds, but usually arrives); undocumented conventions cost correctness — a confident, wrong number from data that looks perfect.

Bottom line

Enrich a low-signal schema and the cheapest model matches a frontier model's accuracy at ~80% lower price and ~50% less time per question ($0.050 vs $0.269) — database readiness is a property of your data, not the model you pick.

Method. Small synthetic benchmark — a deliberately obscure 11–12 table schema, gold answers computed from the source data, a single trial per cell. Numbers are directional, not a leaderboard; effect sizes were consistent across model tiers. Some per-model cells (marked —) were not run.
